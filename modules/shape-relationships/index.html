shape-relationships

Data shapes can be connected in various ways and since node and property shapes are both data shapes, combinations may also be possible between these different types. A visualization should cover all possible combinations of data shape relationship in a consistent manner for an optimal user experience.

This module contains an example covering the following data shape relationships:

  • a node shape referring with an exclusive disjunction to one node and one property shape
  • a property shape referring with a disjunction to two data shapes
  • a property shape referring with a reuse relationship from its object to a node shape
  • a property shape referring with a reuse relationship from its object to a property shape

Example

The example is available here

Description

The example contains a fictive specification of DCAT.

According to an exclusive disjunction constraint, a dcat:Distribution should either have exactly one property dct:rights of class dct:RightsStatement or be compliant to the ex:licenseShape, i.e. have exactly one property dct:license of class dct:LicenseDocument.

According to a disjunction constraint, a dcat:Distribution may have up to three `schema:image properties either of class schema:ImageObject or schema:URL. (remark: the latter one is actually a datatype and not a class, but this example comes from the DCAT application profile for data portals in Switzerland.)

A dcat:Distribution should have at least one dct:publisher property of class foaf:Agent of which additionally at least one object must comply with the node shape ex:agentShape.

A dcat:Distribution must have at least one but maximum two ex:masterRecord properties whose object must comply with a property shape stating that at least one ex:status property of the master record must have the value ex:statusApproved as object.
